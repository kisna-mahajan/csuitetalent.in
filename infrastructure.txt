This document explains how the website is hosted, how the domain works, and

────────────────────────────────────────────────────────────────────────────────
  PART 1 — WHERE THE WEBSITE LIVES (HOSTING)
────────────────────────────────────────────────────────────────────────────────

The website is a collection of plain HTML, CSS, and JS files. There is no
server, no database, and no backend. The files are served directly from GitHub.

  ┌─────────────────────────────────────────────────────────┐
  │  YOUR COMPUTER                                          │
  │                                                         │
  │  You write/edit HTML, CSS, JS files locally             │
  │  and push them via Git to GitHub                        │
  └─────────────────────────┬───────────────────────────────┘
                            │  git push origin main
                            ▼
  ┌─────────────────────────────────────────────────────────┐
  │  GITHUB (github.com)                                    │
  │                                                         │
  │  Repository: kisna-mahajan/csuitetalent.in              │
  │  Branch:     main                                       │
  │                                                         │
  │  GitHub Pages reads the files from the main branch      │
  │  and serves them as a static website automatically.     │
  │  Default URL: kisna-mahajan.github.io/csuitetalent.in   │
  └─────────────────────────────────────────────────────────┘

Every time you push to the main branch, GitHub Pages automatically deploys
the updated files within ~1-2 minutes. No manual deployment step needed.

────────────────────────────────────────────────────────────────────────────────
  PART 2 — HOW THE DOMAIN WORKS (DNS)
────────────────────────────────────────────────────────────────────────────────

A domain name is just a human-readable label that points to an IP address.
DNS (Domain Name System) is the phone book that translates the label to an
address. Cloudflare manages the DNS for csuitetalent.in.

  ┌─────────────────────────────────────────────────────────┐
  │  DOMAIN REGISTRAR                                       │
  │  (e.g. GoDaddy / Namecheap / Google Domains / etc.)    │
  │                                                         │
  │  Owns the rights to "csuitetalent.in"                   │
  │  Nameservers are pointed to Cloudflare                  │
  └─────────────────────────┬───────────────────────────────┘
                            │  "Ask Cloudflare about DNS"
                            ▼
  ┌─────────────────────────────────────────────────────────┐
  │  CLOUDFLARE (cloudflare.com)                            │
  │                                                         │
  │  Acts as the DNS manager for csuitetalent.in.           │
  │  Holds two key records:                                 │
  │                                                         │
  │  A / CNAME record → points csuitetalent.in              │
  │                      to GitHub Pages servers            │
  │                                                         │
  │  MX records        → points email @csuitetalent.in      │
  │                      to Cloudflare Email Routing        │
  │                                                         │
  │  Cloudflare also acts as a CDN and security layer       │
  │  (DDoS protection, caching, HTTPS certificates)         │
  └─────────────────────────────────────────────────────────┘

The CNAME file in the root of the repository (containing "csuitetalent.in")
tells GitHub Pages which custom domain to respond to.

When a visitor types "csuitetalent.in" in their browser:

────────────────────────────────────────────────────────────────────────────────
  PART 4 — HOW EMAIL GETS TO YOUR GMAIL
────────────────────────────────────────────────────────────────────────────────

When someone submits the contact form:

  Visitor fills form on csuitetalent.in/pages/contact.html
       │
       │  1. Browser POSTs form data to FormSubmit.co
       ▼
  FormSubmit.co (formsubmit.co)
       │
       │  2. FormSubmit composes an email and sends it
       │     TO: leadership@csuitetalent.in
       ▼
  The Internet (SMTP email routing)
       │
       │  3. "Where do I deliver mail for @csuitetalent.in?"
       │     Looks up MX records for csuitetalent.in in DNS
       ▼
  Cloudflare DNS
       │
       │  4. MX records point to Cloudflare Email Routing
       ▼
  Cloudflare Email Routing
       │
       │  5. Rule: leadership@csuitetalent.in
       │            → forward to → csuitetalent.in@gmail.com
       ▼
  Gmail (google.com)
       │
       │  6. Delivers to your Gmail inbox
       ▼
  You receive the enquiry email ✓
